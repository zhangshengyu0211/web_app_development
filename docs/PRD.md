# 產品需求文件 (PRD) - 任務管理系統

## 1. 專案概述
- **背景與動機**：現代人日常工作繁雜，需要一個簡單輕量的工具來記錄與追蹤待辦事項，確保工作不遺漏，提升生活與工作效率。
- **目標用戶**：需要管理日常工作、追蹤個人待辦事項的個人用戶。
- **核心價值主張**：提供一個直覺且易用的任務管理平台，讓使用者能快速新增、管理並過濾任務，減少記憶負擔，專注於完成工作。

## 2. 功能需求
系統涵蓋以下核心功能，各項功能的使用者故事 (User Story) 如下：

1. **新增任務**
   - 作為 **個人用戶**，我希望 **能夠輸入任務名稱並新增任務**，以便 **記錄我需要完成的工作**。
2. **標記任務完成**
   - 作為 **個人用戶**，我希望 **能夠將已做完的任務標記為「完成」狀態**，以便 **我能清楚掌握哪些工作已經處理完畢**。
3. **刪除任務**
   - 作為 **個人用戶**，我希望 **能夠刪除不再需要或輸入錯誤的任務**，以便 **保持任務清單的簡潔與正確性**。
4. **顯示所有任務清單**
   - 作為 **個人用戶**，我希望 **能夠預設看到我所有的任務**，以便 **整體檢視我的工作項目**。
5. **依完成狀態篩選**
   - 作為 **個人用戶**，我希望 **能夠依據「全部」、「未完成」、「已完成」來快速篩選任務清單**，以便 **我能更專注於當下需要處理的事項或檢視已完成的成果**。

## 3. 非功能需求
- **技術架構考量**：
  - 架構設計需支援輕量快速的網頁應用開發。若沿用原有技術棧，可使用 Flask 搭配 Jinja2 及 SQLite 進行實作。
  - UI 設計需具備基本的回應式設計 (Responsive Web Design)，以確保在不論桌機或手機上都有良好的操作體驗。
- **效能與操作體驗**：
  - 介面操作需簡單直覺（例如點擊 Checkbox 即可標記完成）。
  - 對於非法輸入（如空白任務）需給予適當提示。

## 4. MVP 範圍 (Minimum Viable Product)
- **Must Have (必須有)**：
  - 核心任務操作：新增任務、標記完成、刪除任務。
  - 頁面顯示：任務清單總覽與狀態篩選機制（全部／已完成／未完成）。
- **Should Have (應該有)**：
  - 任務建立時間紀錄。
  - 基本防呆機制（阻擋空任務名稱的建立）。
- **Nice to Have (加分項)**：
  - 任務截止期限設定。
  - 優先級設定（高、中、低）。
  - 進階動畫回饋（如刪除時有淡出過渡效果）。
